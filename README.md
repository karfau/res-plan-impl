## Why an extra repo?
In it's simplest approach you of course do not need an repository for it.

But since we are learning together how to do it, it is important to share our learnings and grow together.

Also when creating the artifacts (mark down files and potentially additional resources), 
which are the result of one step and the input for the next step, organizing where to put them 
without having to push into a repo that has tons of github actions on push.

It can also be helpful to organize and keep track of multiple plans for either an epic or an initiative,
without poluting the git history of a repo.

This repo should be usable as a template, it should only contain the things required to get you started.
You can either create a branch to get started, or go to the GitHub UI and create a new repo using this one as a template.
